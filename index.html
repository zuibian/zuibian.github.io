安全聊天室项目文档

RSA + AES-GCM 加密 · PyQt5客户端 · Python服务端

一、项目目录结构

chat_project/
├── common/
│ ├── __init__.py
│ ├── crypto.py # 加密相关：AES、RSA、密钥生成

│ ├── __init__.py

二、common模块 — 公共部分

1. crypto.py 加密函数

  • enc_msg(msg, key) — AES-256-GCM加密，返回包含 nonce、密文、tag 的字典
  • dec_msg(data, key) — 解密并验证GCM标签，解密失败抛异常
  • rsa_enc(pub_pem, data) — RSA公钥加密（OAEP填充）
  • rsa_dec(priv_pem, data) — RSA私钥解密
  • gen_rsa() — 生成RSA-2048密钥对，返回PEM格式字符串
  • check_keys() — 检查密钥文件是否存在，不存在则自动生成

注：enc_with_pwd/dec_with_pwd 函数虽然在代码里但登录流程没用上，忽略。

2. protocol.py 消息类型

常量值说明
TYPE_KEY_EXCHANGE'key_exchange'客户端发送加密的会话密钥
TYPE_KEY_EXCHANGE_OK'key_exchange_ok'服务端确认密钥交换完成
TYPE_RECV_MESSAGE'recv_message'服务端转发消息给接收方
TYPE_ONLINE_USERS'online_users'服务端返回在线用户列表
TYPE_KICK'kick'踢出通知（异地登录）
TYPE_CLIENT_BYE'client_bye'客户端退出通知
CMD_LOGIN'login'登录命令
CMD_SEND_MESSAGE'send_message'发送消息命令
CMD_GET_ONLINE_USERS'get_online_users'请求在线用户列表

3. config.py 配置项

  • SERVER_HOST = '127.0.0.1'
  • SERVER_PORT = 9999
  • RSA_PRIVATE_KEY_FILE = "Own_key.Encryption"
  • RSA_PUBLIC_KEY_FILE = "Public_key.Encryption"
  • TIMEOUT_SECONDS = 60 — 服务端socket超时时间

三、client模块 — 客户端

1. main.py 启动流程

  • 调用 load_pub_key() 读取RSA公钥文件
  • os.urandom(32) 生成随机AES会话密钥
  • conn_server() 连接服务端
  • do_key_ex() 用RSA公钥加密会话密钥并发给服务端
  • 用户输入用户名和密码
  • enc_msg(f"{user}:{pwd}", session_key) 加密登录凭证
  • send_login_info() 发送登录请求，接收返回的token
  • 登录成功后调用 start_ui() 启动聊天窗口

2. network.py 网络函数

  • conn_server() — 创建socket连接服务器
  • do_key_ex(sock, pub_key, session_key) — RSA加密session_key并发送，等待确认
  • send_login_info(sock, encrypted_creds) — 发送加密的登录信息，返回服务端响应
  • send_raw_data(sock, data) — 发送原始数据

3. ui.py 聊天窗口 (ChatWin)

  • 继承 QMainWindow，使用 pyqtSignal 处理跨线程消息更新
  • 界面组件：消息区域(QTextEdit)、用户下拉框(QComboBox)、输入框(QLineEdit)、发送按钮、退出按钮
  • send_msg() — 获取目标用户和消息内容，AES加密后发送 send_message 命令
  • get_users() — 被QTimer触发（间隔1秒），请求在线用户列表
  • start_recv() — 独立线程循环接收数据：
    • type == 'recv_message' → 解密并显示消息
    • type == 'online_users' → 更新下拉框和状态栏
    • type == 'kick' → 显示被踢提示并退出
  • exit_chat() — 发送 client_bye，关闭socket，退出界面

四、server模块 — 服务端

1. main.py 服务端入口

  • 创建socket，绑定 127.0.0.1:9999，开始监听
  • 循环accept，每个客户端新建线程执行 ClientHandler(sock, addr).handle()

2. handler.py ClientHandler类

  • handle() — 主循环，先处理密钥交换，然后处理登录、消息等命令
  • do_key_ex(data) — 读取RSA私钥，解密获得AES会话密钥，保存在 self.key
  • do_login(data) — 解密登录凭证，调用 check_pwd() 验证，成功则生成token并存储会话信息
  • send_msg(to_user, encrypted) — 用发送方密钥解密消息，再用接收方密钥重新加密，转发给目标用户
  • send_users() — 返回当前所有在线用户名
  • cleanup() — 断开连接时从 online_users、session_to_user、user_session_key 中移除记录
  • 踢出逻辑：登录时如果用户已在 online_users 中，给旧socket发送 kick 消息

3. storage.py 数据存储

  • USER_DB 字典：用户名 → SHA256(密码)
  • 预置账号：alice/123456、bob/123456、charlie/password
  • online_users = {} — 用户名 → socket对象
  • session_to_user = {} — token → 用户名
  • user_session_key = {} — 用户名 → AES会话密钥
  • check_pwd(user, pwd) — 验证密码
  • add_user(user, pwd) — 添加新用户

五、start.py 启动脚本

  • python start.py server — 启动服务端
  • python start.py client — 启动客户端
  • 自动设置项目根目录到sys.path，方便模块导入

六、通信流程

客户端 服务端
|---- ① key_exchange (RSA加密session_key) ---->|
|<---- ② key_exchange_ok ----------------------|
|---- ③ login (AES加密 user:pass) ------------>|
|<---- ④ {code:0, session: token} -------------|
|---- ⑤ get_online_users --------------------->|
|<---- ⑥ online_users 列表 --------------------|
|---- ⑦ send_message (AES加密消息) ----------->|
| [服务端解密 → 用接收方密钥重加密] |
|<---- ⑧ recv_message (转发加密消息) -----------|
|---- ⑨ client_bye --------------------------->|
+--- 连接关闭

七、运行方式

  • 安装依赖：pip install pycryptodome pyqt5
  • 首次运行会自动生成RSA密钥对（Own_key.Encryption 和 Public_key.Encryption）
  • 先启动服务端：python start.py server
  • 再启动客户端：python start.py client
  • 可以同时启动多个客户端测试聊天
