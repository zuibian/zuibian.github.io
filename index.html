📋 项目概述

本项目实现了一个基于 TCP Socket 的端到端加密聊天室，采用 AES-256-GCM 算法保障消息机密性与完整性，PBKDF2 进行密钥派生。客户端使用 PyQt5 构建图形界面，服务端支持多用户并发、在线状态管理及异地登录踢出机制。

🏗️ 核心架构设计

1️⃣ 客户端 (client.py + ui.py)

  • 认证流程： 用户名/密码 → PBKDF2派生密钥 → AES-GCM加密凭证 → 发送服务端验证
  • 会话管理： 登录成功后获取服务端下发的 session_token，后续所有请求需携带该令牌
  • UI线程与网络线程分离： 接收线程独立运行，避免界面卡顿
  • 定时拉取在线列表： 每3秒通过 QTimer 发送 get_online_users 请求

2️⃣ 服务端 (server.py)

  • 用户认证： SHA256哈希存储密码，验证后生成唯一 session_token
  • 消息路由： 根据 to 字段查找目标用户socket，直接转发消息
  • 并发处理： 每个客户端独立线程，使用全局字典维护在线用户与session映射
  • 踢出机制： 同一账号重复登录时，服务端向旧连接发送 kick 消息

3️⃣ 加密模块 (crypto_aes.py)

函数参数说明
encrypt_gcm(msg, password)明文消息, 主密钥随机生成salt+nonce，PBKDF2派生密钥，返回base64编码的加密字典
decrypt_gcm(encrypted, password)加密字典, 主密钥反解salt+nonce，派生密钥后解密并验证GCM tag

📡 通信协议定义

🔹 客户端 → 服务端

{ "commond": "login", // 登录命令 "data": { // AES-GCM加密后的凭证 "salt": "xxx", "nonce": "xxx", "ciphertext": "xxx", "tag": "xxx" } } { "commond": "send_message", // 发送消息 "session": "uuid+timestamp", "data": { "to": "目标用户", "msg": "消息内容(明文传输，生产环境应二次加密)" } } { "commond": "get_online_users", // 获取在线用户 "session": "uuid+timestamp" } { "type": "client_bye", // 退出通知 "session": "uuid+timestamp" }

🔹 服务端 → 客户端

{ "code": 0, // 登录响应 (0成功 1密码错误 2异常) "session": "xxx", "msg": "登录成功" } { "type": "recv_message", "from": "发送者", "msg": "消息内容" } { "type": "online_users", "users": ["alice", "bob"] } { "type": "kick", "msg": "您在其他地方登录" }

🛡️ 安全机制说明

  • 传输加密： 登录凭证使用 AES-256-GCM 加密，防止中间人窃听
  • 密钥派生： PBKDF2 + 随机盐 (100,000次迭代) 抗暴力破解
  • 完整性校验： GCM模式自带认证标签，防篡改
  • 会话令牌： UUID + 时间戳，防重放攻击
  • 异地登录防护： 服务端检测重复登录主动踢出旧连接
  • ⚠️ 生产环境改进点： 消息内容应使用临时会话密钥加密，而非明文传输
